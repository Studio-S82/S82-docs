Requirements

Ensure all dependencies are installed and running before S82ItemUpgrade in your server.cfg.

Dependency Version Notes
FiveMBuild 6000+Required
ox_inventoryLatestItem management & images
ox_libLatestNotifications
ESX / QB-Core / QBoxAnyAuto-detected, one required
ox_targetLatestOptional — prop interaction
ox_lib and ox_inventory must be ensured BEFORE S82ItemUpgrade in server.cfg or notifications and images will not work.

Installation

1

Download & Extract

Download the resource from your Tebex S82 purchase page and extract the zip file. You will get a folder named S82ItemUpgrade.

2

Place in Resources

Copy the S82ItemUpgrade folder into your server's resources directory.

3

Add to server.cfg

# Make sure dependencies come first
ensure ox_inventory
ensure ox_lib
ensure S82ItemUpgrade
4

Configure

Open config.lua and fill in your settings. See the Configuration section for details.

5

Restart Server

Run restart S82ItemUpgrade in the server console or do a full server restart.

Notify Settings

Config.Notify = {
    duration = 5000,         -- ms
    position = 'center-left', -- top | top-right | bottom-right | center-left ...
    title    = 'S82 Upgrade', -- shown above each notification
}

Server Banner

Displays your server logo inside the UI below the result slot. Clicking the banner resets the UI without closing it.

Config.ServerBanner = {
    enabled = true,
    url     = 'https://your-image-link.png', -- direct image URL
    alt     = 'Your Server Name',
}
Recommended image size
250 × 250 px square (PNG with transparent background looks best on the dark UI). Supports PNG and JPG.

Shield Card

When placed in the shield slot, this item protects the upgrade item on failure. The shield is always consumed when used.

Config.ShieldItem = {
    item   = 'protection_card', -- ox_inventory item name
    label  = 'Protection Card',
    amount = 1,
}

Discord Webhook

Every upgrade event is sent as a Discord embed to your admin channel in real time.

Config.WebhookURL         = 'https://discord.com/api/webhooks/...'
Config.WebhookName        = 'S82 Upgrade | Log'
Config.WebhookAvatar      = 'https://your-avatar.png'
Config.WebhookLogSuccess  = true  -- log successful upgrades
Config.WebhookLogFail     = true  -- log failed upgrades
Config.WebhookLogExploit  = true  -- log detected exploits

What gets logged

Event Color Fields included
Success🟢Player, Discord ID, item, result, bonuses, shield
Fail🔴Player, Discord ID, item, bonuses, shield
Exploit🔴Player, reason, raw data payload

Recipes

Recipes define every item that can be upgraded. Add as many as needed inside Config.Recipes in config.lua.

Field Type Description
idstringox_inventory item name dragged into upgrade slot
resulttable{ item, amount } — reward on success
craftTimenumberDuration of upgrade animation (ms)
baseLucknumberBase success rate (%)
maxLucknumberMax rate including all bonuses (%)
shieldablebooleanAllow shield card on this recipe
failReturntable?Optional item returned to player on failure
requiredItemstable[]Mandatory materials { item, amount, label }
optionalItemstable[]Bonus materials { item, amount, label, luckBonus }

Example Recipe

{
    id          = 'weapon_pistol',
    label       = 'Pistol Upgrade',
    result      = { item = 'weapon_pistol_mk2', amount = 1 },
    craftTime   = 6000,
    baseLuck    = 35,
    maxLuck     = 65,
    shieldable  = true,
    -- failReturn = { item = 'pistol_scrap', amount = 1 },
    requiredItems = {
        { item = 'metalscrap', amount = 5, label = 'Metal Scrap' },
        { item = 'steel',      amount = 3, label = 'Steel' },
    },
    optionalItems = {
        { item = 'gun_oil', amount = 1, label = 'Gun Oil', luckBonus = 15 },
    },
},

How to Play

1

Go to the Upgrade Table

Walk to any crafting location set in config.lua. Look at the prop and press [E] or interact via ox_target.

2

Drag Your Item

Find your item in the inventory panel on the left. Hold and drag it into the UPGRADE ITEM slot in the center.

3

Add Materials

Drag required materials into their slots. Optionally click bonus materials to activate the luck boost.

4

Shield (Optional)

If the recipe supports shield and you have a protection card, drag it into the shield slot.

5

Upgrade!

Click the UPGRADE button and wait for the result animation.

6

Reset for Next Upgrade

Click the server banner image to instantly reset the entire UI for a new upgrade.

FAQ

Item images not showing?

Check Config.ImagePath and ensure ox_inventory is running. Default path:

Config.ImagePath = 'nui://ox_inventory/web/images/'

Notifications not appearing?

Make sure ox_lib is ensured BEFORE S82ItemUpgrade in server.cfg.

ox_target not working?

Set Config.UseOxTarget = true and confirm ox_target is running.

Webhook not receiving messages?

Verify the webhook URL is correct and the Discord bot has permission to post in that channel.

Console says update available?

Download the latest version from your Tebex S82 purchase page.

Sound only plays on first upgrade?

This was a bug in v2.0.0, fixed in v2.1.0. Make sure you are using the latest version.
